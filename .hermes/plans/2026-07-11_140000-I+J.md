## I. 前端拆模块（P0）

**现状**: `static/index.html` = 2175 行，所有 JS 内联 + 大段 HTML 模板字符串
**目标**: 拆为 5 个 JS 文件，index.html 仅保留骨架 + `<script>` 加载

### 模块划分

| 文件 | 内容 | 行数 |
|------|------|------|
| `static/js/api.js` | API 封装、Auth 状态、fetch 拦截器、Auth 帮助函数 | ~80 |
| `static/js/ui.js` | Dark mode、Keyboard shortcuts、Toast、Router、UI 工具函数 | ~100 |
| `static/js/chat.js` | mdToHtml、addChatBubble、sendChatMessage、renderMessage、SSE 处理 | ~350 |
| `static/js/sidebar.js` | 侧边栏渲染、会话列表、搜索过滤、Search history、Message search | ~250 |
| `static/js/tabs.js` | 所有 Tab 渲染函数（dash/status/knowledge/sessions/skills/settings/MCP/reports/scheduler/admin） | ~1100 |
| `static/index.html` | 仅保留 HTML 骨架 + `<script>` 标签 | ~40 |

### 风险与策略
- 全局函数引用：所有函数挂 `window`（已如此），拆后保持同一模式
- HTML 模板中的嵌入 JS：模板字符串接受全局变量，拆到 tabs.js 后全局变量仍在
- 加载顺序：api.js → ui.js → sidebar.js → chat.js → tabs.js

## J. Docker 部署（P1）

**目标**: 单命令 `docker compose up` 启动 LingShu + SearXNG

### 文件

| 文件 | 内容 |
|------|------|
| `Dockerfile` | 多阶段构建：build → slim runtime |
| `docker-compose.yml` | LingShu 服务 + SearXNG 搜索服务 |
| `.dockerignore` | 排除 build artifacts、venv、.git |

### Dockerfile 设计

```dockerfile
# Stage 1: Build
FROM python:3.11-slim AS builder
WORKDIR /app
COPY pyproject.toml .
RUN pip install --no-cache-dir build && python -m build --wheel

# Stage 2: Runtime
FROM python:3.11-slim
WORKDIR /app
COPY --from=builder /app/dist/*.whl .
RUN pip install --no-cache-dir *.whl && rm *.whl
COPY src/agent_harness/static/ /app/static/
EXPOSE 8788
CMD ["agent-harness", "serve"]
```

### 验证
```bash
docker compose build --no-cache 2>&1 | tail -5
docker compose up -d
curl http://127.0.0.1:8788/health
docker compose down
```
